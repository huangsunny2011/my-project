## 🎯 專案特定提示詞

### 設置專案上下文

在開始新功能前，先設置上下文：

```
@workspace 我將開始開發 [功能名稱]。請先：
1. 分析當前代碼庫相關部分
2. 識別相關的文件和模組
3. 說明當前的架構模式
4. 建議實作方式
5. 列出需要注意的事項
```

### 功能實作流程

```
步驟 1: 規劃
@workspace 為 [功能] 創建實作計劃，包含需要修改的文件和步驟

步驟 2: 實作
@workspace 實作 [具體子功能]

步驟 3: 測試
@workspace 為剛實作的功能生成測試

步驟 4: 文檔
@workspace 為新功能更新文檔
```
